### 📌개요      
1️⃣ PostgreSql에서 Json 데이터는 텍스트의 형태로 저장 할 수도 있지만, RFC7159에 정의된 Json Spec(문법)에 상응하는 json, jsonb 두개의 데이터 타입을 지원합니다.      
2️⃣ 또한, JSON 전용함수와 연산자 역시 제공 되고 있습니다.      
3️⃣ RFC7159에 정의된 Json Spec(문법)에 따르면 Json-text 는 utf-8 인코딩을 권고하여 database 인코딩 설정이 utf-8이 아닌경우에 인코딩 실패가 발생할수 있습니다.       
하지만 반대로 utf-8인코딩에서 표현할수 없었던 문자를 취급할수도 있습니다.      
4️⃣ RFC7159에 정의된 Json Spec(문법)에 따라 \uXXXX와 같은 유니코드 이스케이프 시퀀스를 포함할 수 있도록 허용합니다.      
      
### 📌json Data type      
 1️⃣ json-text 그 자체를 대상으로 합니다. (사실상 text)      
 2️⃣ json-text 그 자체를 저장하기 때문에 토큰 사잉의 공백과 같은 의미없는 문자도 같이 취급됩니다.      
 3️⃣ 또한, 입력된 json-text 내 객체(Object)의 키 순서를 보존 하고 중복된 키 존재시 모든 키/값 쌍이 유지 됩니다.       
 (ps. pg 처리 함수 에서는 중복 키/값 존재시 마지막 값을 작동값으로 간주 합니다.)      
 👉 json spec 에 맞기만 하면 해당 데이터를 저장할수 있는 장점이 있지만 처리함수와 같이 json을 처리할때는 해당 text를 해석해야만 하기에 효율적이지는 못합니다.      
 👉 데이터 베이스 인코딩과 관계없이 \uXXXX 의 유니코드 이스케이프 시퀀

### 📌 jsonb Data type      
 1️⃣ json-text를 분해된 이진 형식(decomposed binary format)으로 저장 합니다.      
 2️⃣ jsonb data Type에 따라 공백을 저장하지도 않고 json-text 내 객체(Object)의 순서도 보존하지 않으며, 입력에 중복된 객체 키 존재시 마지막 키/값만 유지 합니다.      
 👉 jsonb 타입으로 변환, 저장시 약간의 지연이 있지만 reparsing이 필요 없고 indexing을 지원하여, json을 처리할때는 엄청나게 효율적 입니다.
