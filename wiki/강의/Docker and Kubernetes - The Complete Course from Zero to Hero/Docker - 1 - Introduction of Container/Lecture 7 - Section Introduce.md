이 섹션에서는 컨테이너에 대해 큰 그림에서 개요를 살펴보고, <u>왜 컨테이너를 사용해야 하는지에 대한 동기(필요성)</u>를 논의하겠습니다.   
  
먼저 <u>“우리는 왜 처음부터 컨테이너를 사용해야 할까?”</u> 라는 질문을 탐구하는 것으로 시작합니다.  
컨테이너를 사용할 때와 사용하지 않을 때 배포가 어떻게 달라질 수 있는지 빠르게 비교한 뒤, <u>컨테이너를 사용할 때 얻는 핵심 이점</u>들을 살펴보겠습니다.  
  
그다음 컨테이너와 가상 머신을 비교합니다.   
두 기술의 차이점을 강조하고, <u>각각을 언제 사용하는 것이 좋은지</u>도 논의하겠습니다.   
컨테이너와 가상 머신은 서로 대체 관계(둘 중 하나만 써야 하는 관계)가 아닙니다.   
즉, 컨테이너의 사용 사례도 있고 가상 머신의 사용 사례도 따로 있으며, <u>각각의 차이와 언제 무엇을 쓰는지가 중요</u>합니다.  
  
마지막으로 Docker 시스템에서 설치되고 실행되는 <u>가장 중요한 Docker 구성요소</u>들을 다룰 것입니다.   
각 구성요소의 목적과, Docker에서 흔히 하는 작업들을 실행할 때 <u>이 구성요소들이 서로 어떻게 상호작용하는지</u> 이해해보겠습니다.   
이를 통해 Docker 시스템을 더 잘 이해하고, <u>어떤 구성요소가 어떤 기능을 담당하는지</u> 파악하는 데 도움이 되길 바랍니다.
